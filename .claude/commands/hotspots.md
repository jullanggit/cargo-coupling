# Hotspots Command

リファクタリング優先度の高いモジュールを特定します。

## 使用方法

```
/hotspots [path]
```

## 実行内容

1. カップリング分析を実行
2. 各モジュールのホットスポットスコアを計算
3. 優先度順にリスト表示

## スコア計算基準

| 要因 | 重み | 説明 |
|------|------|------|
| 問題数 | ×30 | 関連するCoupling Issueの数 |
| カップリング数 | ×5 | 入出力の依存関係数 |
| Health: Critical | +50 | 健全性がCritical |
| Health: Needs Review | +20 | 健全性がNeeds Review |
| 循環依存 | +40 | 循環依存に含まれる |

## 実行手順

```bash
# 分析実行
cargo run --release -- coupling --ai $ARGUMENTS

# 結果を解析してホットスポットを特定
```

## 出力形式

以下の形式でホットスポットを報告してください：

```
## Refactoring Hotspots

### 1. [モジュール名] (スコア: XX)
- **問題**: [問題の種類と数]
- **カップリング**: 入力 X / 出力 Y
- **推奨アクション**: [具体的な改善提案]

### 2. [モジュール名] (スコア: XX)
...
```

## 分析観点

1. **なぜこのモジュールが問題か**
   - どの問題タイプが検出されているか
   - カップリングパターンの特徴

2. **改善の優先度**
   - 影響範囲（Blast Radius）
   - 改善難易度
   - 期待される効果

3. **具体的なリファクタリング提案**
   - インターフェース分離
   - 依存関係の逆転
   - モジュール分割
   - Facade パターン適用

## 注意事項

- スコアが0のモジュールは問題なしとして扱う
- 上位5件を重点的に分析
- 循環依存は最優先で対処を推奨
